# 사용자 사진 등록 시스템 (User Photo Registration System)

본 프로젝트는 사용자의 얼굴 사진(정면, 좌측면, 우측면)을 등록하고 보안 검사 및 품질 검증을 거쳐 로컬 DB에 저장하는 캡스톤 디자인 프로젝트입니다.

---

## 🛠 Tech Stack
- **Frontend**: React, Vite, Tailwind CSS
- **Backend**: Node.js, Express
- **Database**: SQLite3 (`capstone.db`)

---

## 📋 주요 기능 및 설계 결정 사항

### 1. 3단계 등록 프로세스 (UX Flow)
사용자 편의성과 데이터 관리를 위해 등록 과정을 3단계로 분리했습니다.
- **Step 1 (모드 선택)**: '신규 등록'과 '추가 등록' 중 선택. 
- **Step 2 (ID 입력)**: 사용자의 고유 ID(`user_id`)를 직접 입력합니다. 이후 사진을 추가 등록하여 얼굴 벡터를 업데이트할 수 있는 확장성을 고려한 설계입니다.
- **Step 3 (사진 업로드)**: 좌측, 정면, 우측 사진 3장을 업로드 및 검증합니다.

### 2. 사진 검증 시뮬레이션
제출된 사진에 대해 다음 3단계 검증 프로세스를 시뮬레이션합니다.
1. **파일 유효성 검사**: 포맷 및 크기 체크
2. **품질 검증**: 조도 및 선명도 체크
3. **얼굴 정렬**: AI를 통한 특징점 정렬

### 3. 효율적인 데이터 저장 방식
- **이미지 저장**: DB에 바이너리(BLOB)로 직접 저장하는 대신, `/server/images/{user_id}/` 폴더에 파일로 저장하여 DB 용량을 최적화했습니다.
- **DB 데이터**: SQLite(`capstone.db`) 파일에 `user_id`와 이미지 파일의 **상대 경로**만 저장합니다.
- **기본 키(ID)**: 시스템 내부적인 데이터 정합성과 검색 속도 최적화를 위해 자동 증가하는 일련번호(`id`) 필드를 추가했습니다.

## 📂 프로젝트 구조
- `src/`: React 프론트엔드 소스 코드
- `server/`: Express 백엔드 서버 및 DB 설정
  - `index.js`: API 서버 로직 (`/api/register`, `/api/users`)
  - `db.js`: SQLite DB 연동 및 테이블 관리
  - `capstone.db`: SQLite 데이터베이스 파일 (실제 데이터 저장소)
  - `images/`: 업로드된 실제 이미지 파일이 저장되는 곳

## 🚀 실행 방법

### 1. 패키지 설치 (최초 1회)
```bash
# 프론트엔드 의존성 설치
npm install

# 서버 의존성 설치
cd server
npm install
cd ..
```

### 2. 서버 및 앱 실행
**터미널 1 (백엔드 서버):**
```bash
cd server
node index.js
```
**터미널 2 (프론트엔드):**
```bash
npm run dev
```
이후 `http://localhost:5173`에서 확인할 수 있습니다.
