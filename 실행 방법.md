# 실행 방법

본 프로젝트는 두 개의 독립적인 시스템으로 구성되어 있습니다.

- **A. 통합 시스템** (`start_app.cmd`로 일괄 실행)
  - Orchestrator (FastAPI, 포트 8080)
  - Evidence Collection Module (Node.js, 포트 13000)
  - Crawl & Takedown Module (FastAPI, 내부 포트 8888 — 외부 접근은 8080 경유)
- **B. 사용자 사진 등록 시스템** (`User photo register system_new/` — 별도 실행)
  - React 프론트엔드 (Vite, 포트 5173)
  - Express 백엔드 (Node.js)

`node_modules/`, 데이터베이스 파일, 업로드 이미지, 로그는 zip에서 제외되어 있으므로 아래 절차에 따라 의존성 설치 후 실행하면 됩니다.

---

## 0. 사전 요구사항

| 항목 | 권장 버전 |
|---|---|
| Python | 3.10 이상 |
| Node.js | 18 LTS 이상 (npm 포함) |
| OS | Windows 10/11 (`start_app.cmd`는 Windows 기준) |

선택 사항:
- `module_vector_extract`, `module_vector_match`를 실제로 구동하려면 **Redis** 서버가 필요합니다 (로컬 6379 포트). 통합 데모만 돌릴 경우 필수는 아닙니다.
- `module_vector_extract`는 **InsightFace** 모델 다운로드를 수반하므로 최초 실행 시 인터넷 연결이 필요합니다.

## 1. 의존성 설치 (최초 1회)

### 1-1. Python 의존성

PowerShell에서 프로젝트 루트(`Capstone_260509/`)로 이동한 뒤 가상환경을 만드는 것을 권장합니다.

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1

pip install -r pip\requirements.txt
pip install -r module_vector_match\requirements.txt

# module_vector_extract 는 별도 requirements.txt 가 없으므로 직접 설치
pip install fastapi "uvicorn[standard]" python-multipart opencv-python insightface redis numpy Pillow

# module_crawl_takedown 도 별도 requirements.txt 가 없습니다 — 위 pip\requirements 로 충분
```

### 1-2. Node.js 의존성 — Evidence Collection Module

```powershell
cd "Evidence Collection Module"
npm install
cd ..
```

### 1-3. Node.js 의존성 — User photo register system_new

```powershell
cd "User photo register system_new"
npm install

cd server
npm install
cd ..\..
```

## 2. A. 통합 시스템 실행

프로젝트 루트(`Capstone_260509/`)에서:

```powershell
.\start_app.cmd
```

스크립트가 다음 3개 서비스를 순차로 띄웁니다.

1. Evidence Collection Module 서버 (`Evidence Collection Module/server.js`)
2. Crawl & Takedown 서버 (`module_crawl_takedown/server.py`)
3. Orchestrator (`pip/orchestrator.py`) — 포그라운드 실행

브라우저에서 다음 주소로 접속합니다.

| 화면 | URL |
|---|---|
| 통합 포털 / 사용자 등록 UI | http://localhost:8080/register-ui/ |
| 증거 수집 UI | http://localhost:8080/evidence/index.html |
| 예시 대응 사이트 | http://localhost:8080/example/ |

종료할 때는 Orchestrator 콘솔에서 `Ctrl+C`로 종료한 뒤, 백그라운드로 실행된 나머지 두 서버는 작업 관리자에서 `node.exe` / `python.exe` 프로세스를 종료하거나 PowerShell에서 다음과 같이 정리합니다.

```powershell
Get-Process node, python -ErrorAction SilentlyContinue | Stop-Process
```

### 수동으로 개별 실행 (디버깅용)

```powershell
# 터미널 1 — Evidence
cd "Evidence Collection Module"
node server.js

# 터미널 2 — Crawl & Takedown
cd module_crawl_takedown
python server.py

# 터미널 3 — Orchestrator
cd pip
python orchestrator.py
```

## 3. B. 사용자 사진 등록 시스템 실행 (별도)

`User photo register system_new/`는 자체적으로 동작하는 React + Express 시스템입니다. 통합 시스템과 별개로 실행하려면:

```powershell
cd "User photo register system_new"

# 터미널 1 — Express 백엔드
cd server
node index.js

# 터미널 2 — Vite 개발 서버
cd ..
npm run dev
```

또는 한 번에:

```powershell
cd "User photo register system_new"
npm run dev:all
```

접속: http://localhost:5173

## 4. 데이터베이스 / 저장소 초기화

zip에는 SQLite DB 파일과 업로드 이미지 폴더가 포함되어 있지 않습니다. 코드가 최초 실행 시 자동으로 생성합니다.

| 위치 | 생성 시점 |
|---|---|
| `User photo register system_new/server/capstone.db` | `node index.js` 첫 실행 시 (`db.js`가 스키마 자동 생성) |
| `User photo register system_new/server/images/` | 사진 업로드 시 자동 생성 |
| `module_crawl_takedown/unified.db` | 서버 첫 실행 시 (`core/schema.sql` 적용) |
| `module_crawl_takedown/uploads/`, `images/` | 크롤링/업로드 시 자동 생성 |
| `Evidence Collection Module/database.sqlite` | 서버 첫 실행 시 자동 생성 |
| `pip/match.db`, `pip/data/faces.db` | 첫 실행 시 자동 생성 |

DB 스키마를 수동으로 초기화하려면:

```powershell
# crawl_takedown DB 초기화 (해당 모듈에 init 스크립트가 존재)
cd module_crawl_takedown
python scripts\init_db.py
```

## 5. 자주 발생하는 문제

| 증상 | 원인 / 해결 |
|---|---|
| `ModuleNotFoundError: shared` | `module_vector_extract` 등은 프로젝트 루트에서 실행해야 `shared` 패키지를 import 할 수 있습니다. 또는 `PYTHONPATH`에 루트 추가. |
| `EADDRINUSE :8080 / :13000 / :5173` | 포트가 이미 사용 중. 해당 프로세스를 종료하거나 코드의 포트 번호를 변경하세요. |
| `insightface` 모델 다운로드 실패 | 인터넷 연결 확인. 최초 1회만 다운로드되며, 이후 캐시(`~/.insightface/models/`)에서 로드됩니다. |
| `npm install` 시 `sqlite3` 빌드 오류 | Windows 빌드 도구 부재. `npm install --global windows-build-tools` 또는 Node.js 버전을 LTS로 맞추세요. |
| `start_app.cmd` 실행 시 한글 깨짐 | `chcp 65001` 후 다시 실행하거나, Windows Terminal을 사용하세요. |
